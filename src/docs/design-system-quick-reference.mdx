# Design System Quick Reference for AI Agents

## 🚨 CRITICAL RULES

1. **NEVER hardcode colors** - Always use semantic color classes (`bg-primary`, `text-foreground`, etc.)
2. **NEVER use Tailwind text size/weight classes** - Let semantic HTML handle typography (`<h1>`, `<p>`, etc.)
3. **ONLY use 'Inter' font family** - Already set globally, don't override
4. **ALWAYS use pre-built components** from `/src/app/components/ui/` - Don't build from scratch
5. **ALWAYS use CSS variables** - Enables users to customize via `/src/styles/theme.css`

---

## Color Classes Reference

```tsx
// Backgrounds & Text
bg-background          // Main app background
text-foreground        // Main text color
bg-card                // Card backgrounds
bg-popover             // Dropdown backgrounds

// Actions
bg-primary text-primary-foreground           // Primary buttons/actions
bg-secondary text-secondary-foreground       // Secondary actions
bg-destructive text-destructive-foreground   // Delete/dangerous actions

// States
bg-muted text-muted-foreground    // Disabled/subtle
bg-accent text-accent-foreground  // Hover/highlights
border-border                     // Borders
ring-ring                         // Focus rings

// Charts
bg-chart-1 to bg-chart-5          // Chart colors
```

## Typography Rules

```tsx
// ✅ DO THIS - Use semantic HTML
<h1>Heading 1</h1>      // 56px, Bold, Inter - automatic
<h2>Heading 2</h2>      // 48px, Bold, Inter - automatic
<h3>Heading 3</h3>      // 40px, Bold, Inter - automatic
<h4>Heading 4</h4>      // 32px, Semi-Bold, Inter - automatic
<p>Body text</p>        // 16px, Normal, Inter - automatic
<label>Label</label>    // 14px, Semi-Bold, Inter - automatic
<button>Button</button> // 16px, Medium, Inter - automatic

// If you need CSS variables:
style={{ fontSize: 'var(--text-lg)', fontWeight: 'var(--font-weight-bold)' }}

// ❌ DON'T DO THIS
<h1 className="text-4xl font-bold">  // Wrong - removes design system
<p className="text-sm">               // Wrong - use semantic HTML
```

## ❌ Common Mistakes

```tsx
// ❌ DON'T hardcode colors
<div className="bg-blue-500 text-white">

// ✅ DO use semantic classes
<div className="bg-primary text-primary-foreground">

// ❌ DON'T override typography
<h1 className="text-4xl font-bold">

// ✅ DO use semantic HTML
<h1>Heading</h1>

// ❌ DON'T build components manually
<button className="px-4 py-2 bg-blue-500 rounded">

// ✅ DO use design system components
<Button>Click</Button>

// ❌ DON'T use wrong fonts
<p style={{ fontFamily: 'Arial' }}>

// ✅ DO use Inter (already default)
<p>Text</p>
```

## File Locations

- **Design Tokens**: `/src/styles/theme.css` (users edit this to customize)
- **UI Components**: `/src/app/components/ui/*` (import these)
- **Main App**: `/src/app/App.tsx`

---

**Remember**: By using CSS variables and semantic classes, users can customize the entire design system by editing `/src/styles/theme.css` without touching component code.
