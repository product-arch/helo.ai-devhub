# Design System Instructions for AI Agents

This document provides comprehensive instructions for generating UI components that adhere to our design system. All design tokens are defined in `/src/styles/theme.css` and must be referenced through CSS variables or Tailwind CSS classes.

---

## 🎨 Color System

### Available Color Tokens

All colors are defined as CSS variables and exposed as Tailwind classes. **ALWAYS use these semantic color tokens instead of hardcoded colors.**

#### Usage Pattern
```tsx
// ✅ CORRECT - Use Tailwind classes that reference CSS variables
<div className="bg-primary text-primary-foreground">Primary Action</div>
<div className="bg-secondary text-secondary-foreground">Secondary Action</div>
<div className="border border-border">Container</div>

// ❌ INCORRECT - Never hardcode colors
<div className="bg-blue-500 text-white">Action</div>
<div style={{ backgroundColor: '#3186DF' }}>Action</div>
```

#### Primary & Secondary Colors
- `bg-primary` / `text-primary` - Primary brand color (rgba(49, 134, 223, 1))
- `text-primary-foreground` - Text on primary backgrounds (rgba(255, 255, 255, 1))
- `bg-secondary` / `text-secondary` - Secondary actions (rgba(255, 255, 255, 1))
- `text-secondary-foreground` - Text on secondary backgrounds (rgba(20, 24, 27, 1))

#### Surface Colors
- `bg-background` - Main application background (rgba(255, 255, 255, 1))
- `text-foreground` - Main text color (rgba(20, 24, 27, 1))
- `bg-card` - Card/container backgrounds (rgba(255, 255, 255, 1))
- `text-card-foreground` - Text on cards (rgba(20, 24, 27, 1))
- `bg-popover` - Dropdown/tooltip backgrounds (rgba(255, 255, 255, 1))
- `text-popover-foreground` - Text in dropdowns (rgba(20, 24, 27, 1))

#### State Colors
- `bg-muted` / `text-muted` - Disabled/subtle backgrounds (rgba(242, 243, 243, 1))
- `text-muted-foreground` - Secondary text (rgba(103, 110, 115, 1))
- `bg-accent` / `text-accent` - Highlights/hover states (rgba(240, 248, 254, 1))
- `text-accent-foreground` - Text on accents (rgba(49, 134, 223, 1))
- `bg-destructive` - Destructive actions (rgba(237, 28, 52, 1))
- `text-destructive-foreground` - Text on destructive (rgba(255, 255, 255, 1))

#### UI Element Colors
- `border-border` - Default borders (rgba(206, 209, 211, 1))
- `bg-input` - Input field backgrounds (rgba(255, 255, 255, 1))
- `bg-input-background` - Input backgrounds (rgba(255, 255, 255, 1))
- `ring-ring` - Focus ring color (rgba(49, 134, 223, 1))

#### Chart Colors
- `bg-chart-1` / `fill-chart-1` / `stroke-chart-1` - rgba(49, 134, 223, 1)
- `bg-chart-2` / `fill-chart-2` / `stroke-chart-2` - rgba(32, 176, 56, 1)
- `bg-chart-3` / `fill-chart-3` / `stroke-chart-3` - rgba(237, 28, 52, 1)
- `bg-chart-4` / `fill-chart-4` / `stroke-chart-4` - rgba(214, 90, 9, 1)
- `bg-chart-5` / `fill-chart-5` / `stroke-chart-5` - rgba(130, 137, 142, 1)

#### Sidebar Colors
- `bg-sidebar` - Sidebar background (rgba(255, 255, 255, 1))
- `text-sidebar-foreground` - Sidebar text (rgba(75, 79, 83, 1))
- `bg-sidebar-primary` - Sidebar primary (rgba(49, 134, 223, 1))

## 📝 Typography System

### CRITICAL: Font Family Rules

**ALWAYS use 'Inter' font family for ALL text elements.** This is the only font family allowed in this design system.

```tsx
// ✅ CORRECT - Typography handled automatically
<h1>Heading text</h1>
<p>Body text</p>
<button>Button text</button>

// ✅ CORRECT - If you need to manually set font family
<div style={{ fontFamily: "'Inter', sans-serif" }}>Text</div>

// ❌ INCORRECT - Never use other fonts
<div style={{ fontFamily: 'Arial' }}>Text</div>
<p className="font-serif">Text</p>
```

### Typography Scale

**DO NOT use Tailwind's text size classes (text-sm, text-lg, etc.) for typography.** The base typography is automatically applied through CSS.

#### Heading Elements
```tsx
// ✅ CORRECT - Use semantic HTML, styling is automatic
<h1>Display Large</h1>     // 56px / 3.5rem, Bold, Inter
<h2>Display Medium</h2>    // 48px / 3rem, Bold, Inter
<h3>Display Small</h3>     // 40px / 2.5rem, Bold, Inter
<h4>Title</h4>             // 32px / 2rem, Semi-Bold, Inter

// ❌ INCORRECT - Don't override with Tailwind classes
<h1 className="text-4xl font-bold">Heading</h1>
```

#### Body & Small Text
```tsx
// ✅ CORRECT
<p>Body text</p>              // 16px / 1rem, Normal, Inter
<label>Label text</label>     // 14px / 0.875rem, Semi-Bold, Inter
<input placeholder="Text" />  // 16px / 1rem, Normal, Inter
<button>Button</button>       // 16px / 1rem, Medium, Inter

// Special classes
<p className="caption">CAPTION TEXT</p>  // 12px, Normal, Uppercase, Inter
<span className="badge">Badge</span>     // 14px, Semi-Bold, Inter
```

### Font Weight Variables

When you need to apply font weights directly, use these CSS variables:

```tsx
// ✅ CORRECT - Use CSS variables
<p style={{ fontWeight: 'var(--font-weight-normal)' }}>Normal (400)</p>
<p style={{ fontWeight: 'var(--font-weight-medium)' }}>Medium (500)</p>
<p style={{ fontWeight: 'var(--font-weight-semi-bold)' }}>Semi-Bold (600)</p>
<p style={{ fontWeight: 'var(--font-weight-bold)' }}>Bold (700)</p>

// ❌ INCORRECT - Don't use Tailwind font weight classes
<p className="font-bold">Text</p>
```

### Text Size Variables

If you must set custom text sizes, use these CSS variables:

```tsx
// ✅ CORRECT - Use CSS variables
<span style={{ fontSize: 'var(--text-xs)' }}>12px</span>
<span style={{ fontSize: 'var(--text-sm)' }}>14px</span>
<span style={{ fontSize: 'var(--text-base)' }}>16px</span>
<span style={{ fontSize: 'var(--text-lg)' }}>40px</span>
<span style={{ fontSize: 'var(--text-xl)' }}>48px</span>
<span style={{ fontSize: 'var(--text-2xl)' }}>56px</span>
<span style={{ fontSize: 'var(--text-caption)' }}>12px</span>
<span style={{ fontSize: 'var(--text-badge)' }}>14px</span>

// ❌ INCORRECT - Don't use Tailwind text size classes
<span className="text-sm">Text</span>
<span className="text-3xl">Text</span>
```

Use Tailwind's default spacing scale with standard classes:

```tsx
// ✅ CORRECT - Use Tailwind spacing classes

<div className="m-6">Margin 24px</div>

// Common spacing values:
// 0.5 = 2px, 1 = 4px, 2 = 8px, 3 = 12px, 4 = 16px
// 5 = 20px, 6 = 24px, 8 = 32px, 10 = 40px, 12 = 48px
```

## 🔲 Border Radius

Use Tailwind's radius classes that reference the CSS variable `--radius` (8px):

```tsx
// ✅ CORRECT - Use Tailwind radius classes
<div className="rounded-sm">   // calc(var(--radius) - 4px) = 4px
<div className="rounded-md">   // calc(var(--radius) - 2px) = 6px
<div className="rounded-lg">   // var(--radius) = 8px (default)
<div className="rounded-xl">   // calc(var(--radius) + 4px) = 12px

// Or use CSS variables directly
<div style={{ borderRadius: 'var(--radius)' }}>8px radius</div>

// ❌ INCORRECT - Don't hardcode radius values
<div className="rounded-[12px]">Content</div>
<div style={{ borderRadius: '16px' }}>Content</div>
```

## ❌ Common Mistakes to Avoid

### Don't Hardcode Colors
```tsx
// ❌ WRONG
<div className="bg-blue-500 text-white">
<div style={{ backgroundColor: '#3186DF', color: '#FFFFFF' }}>
<button className="bg-red-600">Delete</button>

// ✅ CORRECT
<div className="bg-primary text-primary-foreground">
<Button variant="destructive">Delete</Button>
```

### Don't Override Typography
```tsx
// ❌ WRONG - Don't use Tailwind text classes for font size/weight
<h1 className="text-4xl font-bold">Heading</h1>
<p className="text-sm">Body text</p>
<button className="font-semibold">Action</button>

// ✅ CORRECT - Let semantic HTML handle styling
<h1>Heading</h1>
<p>Body text</p>
<button>Action</button>

// Or use CSS variables when needed
<div style={{ fontSize: 'var(--text-lg)', fontWeight: 'var(--font-weight-bold)' }}>
  Custom text
</div>
```

### Don't Use Wrong Font Families
```tsx
// ❌ WRONG
<p className="font-serif">Text</p>
<p style={{ fontFamily: 'Arial' }}>Text</p>

// ✅ CORRECT - Inter is set globally, no need to specify
<p>Text</p>

// Or if explicitly needed:
<p style={{ fontFamily: "'Inter', sans-serif" }}>Text</p>
```

### Don't Hardcode Border Radius
```tsx
// ❌ WRONG
<div className="rounded-[16px]">
<div style={{ borderRadius: '12px' }}>

// ✅ CORRECT
<div className="rounded-lg">
<div style={{ borderRadius: 'var(--radius)' }}>
```

### Don't Build Components from Scratch
```tsx
// ❌ WRONG - Building custom button
<button className="px-4 py-2 bg-blue-500 text-white rounded-md hover:bg-blue-600">
  Click Me
</button>

// ✅ CORRECT - Use design system Button
import { Button } from './components/ui/button'
<Button>Click Me</Button>
```

## 📋 Quick Reference Checklist

When generating UI components, ensure you:

- [ ] Use semantic color classes (`bg-primary`, `text-foreground`, etc.)
- [ ] Never hardcode color values
- [ ] Use semantic HTML elements (`<h1>`, `<p>`, `<button>`, etc.)
- [ ] Don't override typography with Tailwind text classes
- [ ] Only use 'Inter' font family
- [ ] Import and use design system components from `/src/app/components/ui/`
- [ ] Use Tailwind spacing classes (`p-4`, `gap-6`, `space-y-4`, etc.)
- [ ] Use radius classes (`rounded-lg`, `rounded-md`, etc.)
- [ ] Use CSS variables when direct access is needed
- [ ] Follow component composition patterns

## 🔄 Updating the Design System

To customize the design system, users should edit `/src/styles/theme.css`:

```css
/* Users can update these values */
:root {
  --primary: rgba(49, 134, 223, 1);        /* Change primary color */
  --radius: 8px;                           /* Change default radius */
  --text-lg: 2.5rem;                      /* Change heading sizes */
  --font-weight-bold: 700;                /* Change font weights */
  /* etc. */
}
```

**Important**: Never hardcode values in components. Always reference CSS variables so users can customize the design system by editing the theme file.

## Summary

This design system prioritizes:
1. **Consistency** - Use semantic tokens, not hardcoded values
2. **Flexibility** - Users can customize via CSS variables
3. **Simplicity** - Use pre-built components, avoid reinventing
4. **Typography** - Inter font only, semantic HTML styling
5. **Accessibility** - Proper semantic structure and focus states

Follow these guidelines strictly to ensure all generated UI adheres to the design system and allows users to customize styling through CSS variable updates.
